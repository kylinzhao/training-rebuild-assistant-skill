# Contributing

Thanks for considering a contribution.

## Good contribution types

- improve the skill prompt without making it bloated
- add clearer safety language
- improve screenshot-ingestion guidance
- add reusable examples and report templates
- improve docs for Hermes users

## Contribution guidelines

1. Keep the project **AI native**. Avoid turning it into a giant rule engine.
2. Preserve the **date-isolation rule**. Never mix data across dates.
3. Prefer **clearer prompts and structure** over premature automation.
4. Keep language practical, scientific, and respectful.
5. Do not add personal health data or secrets to this repo.

## Suggested workflow

1. Fork the repo
2. Create a branch: `feat/my-change`
3. Make focused edits
4. Update docs/examples if behavior changes
5. Open a pull request with:
   - the problem
   - the proposed change
   - any trade-offs

## Security and privacy

- Never commit personal health logs, screenshots, API keys, or tokens.
- If a contribution changes data handling, explain privacy implications clearly.

## Pull request checklist

- [ ] The change supports the AI-native design
- [ ] Date isolation is preserved
- [ ] Docs stay internally consistent
- [ ] No personal user data was committed
- [ ] Examples still match actual skill behavior
